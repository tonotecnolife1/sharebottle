# 顧客一覧 リスト/マップ切替 & 管理者フィールド 設計書

## 背景

### 解決したい課題
1. **顧客の担当構造が浅い**: 現状は `cast_id`（担当者）しか無く、管理者（ママ/姉さん）が誰なのか追跡できない
2. **顧客関係性を見る手段が限られる**: 既存の `/mama/map` は「顧客ベース紹介ツリー」しか無い
3. **キャスト側から見る視点が欲しい**: 「あかり担当の全顧客」のような軸で見れない

### 目指す状態
- 顧客一覧ページで **リスト表示 ⇄ マップ表示** を1タップで切替
- マップは **顧客ベース（紹介チェーン）** と **キャストベース（担当者別）** を切替
- 各顧客が「誰の紹介」「誰の管理下」「誰が担当」の3情報をすべて保持

## データモデル

### Customer への追加フィールド

```ts
interface Customer {
  // 既存
  id, store_id, cast_id, name, birthday, job,
  favorite_drink, category, store_memo, created_at,
  referred_by_customer_id,     // 紹介元 (既に追加済み)
  funnel_stage,                 // 既存
  line_exchanged_cast_id,       // 既存
  line_exchanged_at,            // 既存

  // 新規追加
  /** 
   * この顧客を「管理する」ママまたは姉さん。
   * 担当者(cast_id)とは別概念で、上位ポジション。
   * - 担当者がキャスト(help)なら、管理者はその上の姉さんかママ
   * - 担当者が姉さんなら、管理者は本人 or その上のママ
   * - 管理者だけがいるケースも可（担当者割り当て前）
   */
  manager_cast_id: string | null;
}
```

### 既存フィールドとの関係整理

| 概念 | フィールド | 役割 | 例 |
|-----|---------|-----|-----|
| **紹介元** | `referred_by_customer_id` | 誰の紹介で来たか | 田中さま → 高橋さま |
| **管理者** | `manager_cast_id` | 上位ポジション責任者 | ゆき姉さん |
| **担当者** | `cast_id` | 現場で接客する人 | あかり or あやな |

管理者と担当者が同じこともある（姉さんが自分で持つ顧客）。

### Supabase スキーマ（将来）

```sql
alter table customers
  add column manager_cast_id uuid references casts(id);
create index customers_manager_cast_idx on customers(manager_cast_id);
```

```
┌───────────────────────────┐
│ 田中太郎さま     [VIP] 💬 │
```

- `manager_cast_id` が null の場合、リストは「管理者未設定」として別セクションに表示
- マップ（キャストベース）でも「管理者なし」グループを末尾に

## インタラクション

### モード切替の状態保持
- `?view=list|map` と `?map=customer|cast` をクエリに反映
- localStorage にもユーザーの最後の選択を保存し、次回復帰時に復元

### パフォーマンス
- 100人規模までは同期レンダリングでOK
- 1000人規模はvirtualized リスト + 折りたたみ必須（Phase 2）

### モバイル制約
- ツリーのネストが深い時、水平スクロール許容
- 「全展開/全畳み」ボタン
- ピンチズームは実装せず、代わりに「コンパクトモード」トグル

## オープンクエスチョン

- **管理者の自動推論**: 担当者が「姉さん」の時、管理者 = その姉さん？ それとも常に手動？
  - 提案: 担当者が姉さんなら管理者はデフォルトで本人。担当者がキャスト(help)なら、その上の姉さんをデフォルトに。オーバーライド可。
- **管理者の変更履歴**: 誰がいつ管理者を変えたか追跡する？
  - MVP では履歴なし、Phase 4 で検討
- **紹介元との関係**: 紹介元が別の管理者の下だったらどうする？
  - 現状: 管理者は独立したフィールド、紹介チェーンと独立
- **表示上限**: 1000人規模の顧客でマップを描くと重い
  - MVP: 100人まで全展開 / 超えたら折りたたみ + 検索フィルタ必須
