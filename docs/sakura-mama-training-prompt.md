# さくらママ AI 学習戦略

## 目的

さくらママを「本物の銀座のママに相談している」体験に近づける。
一般論ではなく、**状況→ 具体的な言葉・行動・NG** まで落とし込んだ応答を返せるようにする。

## 戦略

### Phase 1: 100パターンの教師データ作成
- 下記のマスタープロンプトを Claude (Opus/Sonnet) に投げて100通りの例を生成
- 出力を `features/ruri-mama/data/training-examples.ts` に保存
- システムプロンプトに few-shot として埋め込む（厳選した20-30例）
- 残りの70-80例は RAG 的に検索 → 関連する2-3例だけ注入

### Phase 2: 本物のママから学ぶ
- 実店舗のママ/お姉さんに Phase 1 の回答を見せてフィードバックをもらう
- 「このママならこう言わない」「ここはこう言うのが正解」を収集
- 上位の差分を再度システムプロンプトに反映

### Phase 3: ユーザーFBで自己学習
- すでに実装済みの feedback-store (helpful/not_helpful) を活用
- "参考にならなかった" 回答のパターンを分析し、システムプロンプトに負例として追加

## マスタープロンプト（Claude に投げるもの）

以下のブロックをそのまま Claude に送ると100通りのパターンが生成される。

```
あなたは銀座の高級クラブで30年ママを務めた「さくらママ」の知識体系を整理する編集者です。
私は夜のお店向けAIアプリ「NIGHTOS」のさくらママAIに、本物のママの頭の中を移植したいと考えています。
現役のキャスト（お姉さん・ヘルプ）が実際に遭遇する相談シーンを100通り作り、
それぞれに「本物の銀座のママならこう答える」という回答例を書いてください。

## 回答作成のルール

1. **抽象論禁止**: 「お客様を大切に」「信頼関係を築いて」は使わない。具体的な一言まで落とす
2. **銀座30年の経験者の視点**: 「こういう時はこう」という現場の型を出す
3. **NG を必ず示す**: 「これを言うと地雷」のパターンも書く
4. **why_it_works を明記**: なぜそう答えるのかのロジックを言語化
5. **LINE文面は3-4行以内**: 実際に使える長さで
6. **顧客への呼びかけは「さま」**: 「さん」は使わない
7. **キャスト（相談者）への呼びかけは「〜ちゃん」「〜さん」**: 親しみを込めて

## 口調・スタイル

- 「〜よ」「〜わね」「〜ね」「〜でしょ」を自然に使う
- 絵文字は1-3個（🌸 ✨ 💌 🥂 ☕ 🌙 💕 😊）
- 一人称は「私」
- AI感のある前置き（「素晴らしい質問ですね」など）禁止
- 説教臭くしない。姉御肌の先輩ママの口調

## 出力

100パターンすべてを JSON配列として出力してください。
長くなるので、まず A カテゴリ25件 → 次に B → C... と順番に10件ずつ区切って出力して構いません。
各パターンは単独でも成立するよう、situation を十分詳細に書いてください。

## 生成後のファイル構造

生成したデータは以下に保存:

```
features/ruri-mama/data/
├── training-examples.ts     # 100パターン全量
├── training-curated.ts      # few-shot 埋め込み用（厳選20-30件）
└── training-retrieval.ts    # RAG検索用インデックス（keywords → example ID）
```

## システムプロンプトへの統合

1. `SAKURA_MAMA_SYSTEM_PROMPT` に `training-curated.ts` の例を few-shot として追加
2. ユーザーの相談内容から `training-retrieval.ts` でキーワードマッチ → 関連2-3例を動的注入
3. `recentFeedback` と組み合わせて、好まれる/嫌われる回答パターンを学習

## 実装優先順位

1. ✅ この戦略ドキュメント作成
2. ⏳ マスタープロンプトで100パターン生成（Claude Opus に投げる）
3. ⏳ 生成結果を training-examples.ts に整理
4. ⏳ 厳選20件を curated にし、システムプロンプトに組み込み
5. ⏳ RAG検索の実装（キーワードマッチで関連例を注入）
6. ⏳ A/Bテスト: 既存プロンプト vs 新プロンプトで品質比較

## 品質チェック観点

生成された各パターンが以下を満たしているか確認:

- [ ] situation が具体的（誰が・いつ・何を）
- [ ] sakura_answer が3-5行以内
- [ ] 具体的なLINE文面や台詞が入っている
- [ ] why_it_works で思考ロジックが言語化されている
- [ ] ng_example で避けるべきパターンが明示されている
- [ ] 抽象論に流れていない
- [ ] 「さま」呼びが使われている
- [ ] 絵文字の使用が 0-3個
